# PRD 最低完备条件与追问优先级

> **用途**：`validate_prd_completeness` skill 的判断依据，也是 Alice 决定是否追问的检查清单。

---

## 一、最低完备条件（Minimum Completeness Criteria）

PRD 必须满足以下条件才能进入下游（UX/Visual/Dev）：

### 1.1 必填字段（P0 - 缺失则阻塞）

| 段落 | 字段 | 判断标准 | 缺失后果 |
|---|---|---|---|
| 1.2 目标用户 | 目标用户是谁 | 非空，非占位符 | 无法确定设计方向 |
| 1.2 目标用户 | 用户当前心智 | 非空，非占位符 | 无法设计情绪体验 |
| 1.3 体验目标 | 情绪目标 | 非空，非占位符 | 无法定义设计基调 |
| 1.3 体验目标 | 核心隐喻 | 非空，非占位符 | 无法生成 Brand DNA |
| 2.1 成功标准 | OMTM | 有指标 + 口径 + 目标值 | 无法验收项目成功与否 |
| 2.4 终端与适配 | 主终端 | 明确单选（Web/Mobile/Landing/Desktop） | 无法确定设计约束 |
| 2.4 终端与适配 | 首要动作 | 非空，非占位符 | 无法确定 CTA 设计 |
| 三、需求清单 | 至少 1 个 P0 功能 | 有模块 + 功能点 + 优先级 + 验收标准 | 无可开发内容 |

### 1.2 强烈建议字段（P1 - 缺失则警告）

| 段落 | 字段 | 判断标准 | 缺失影响 |
|---|---|---|---|
| 1.3 体验目标 | 成功瞬间 | 非空，非占位符 | 缺少"高光时刻"设计指引 |
| 2.3 范围 | In Scope / Out of Scope | 至少各 1 条 | 范围蔓延风险 |
| 2.4 终端与适配 | 信息密度目标 | 非空，非占位符 | 设计密度不确定 |
| 四、核心流程 | 核心用户流程 | 有 Mermaid 图或步骤描述 | 开发理解偏差风险 |
| 五、约束 | 技术栈 | 非空，非占位符 | 技术方案不确定 |

### 1.3 可选字段（P2 - 缺失仅提示）

| 段落 | 字段 | 说明 |
|---|---|---|
| 七、数据与埋点 | 事件列表 | 可由运营后续补充 |
| 八、发布与验证 | 发布策略 | 可在开发完成后补充 |

## 二、追问优先级（Ask Priority）

当 PRD 缺失字段时，按以下优先级追问用户：

### 优先级 1（必须立即追问 - 阻塞下游）

| 缺失字段 | 追问话术 |
|---|---|
| 目标用户是谁 | "这个产品是给谁用的？我需要知道他们的身份才能设计正确的体验。" |
| 用户当前心智 | "用户使用这个产品时是什么状态？焦虑？赶时间？好奇？这决定了我们的设计基调。" |
| 情绪目标 | "你希望用户在使用后感觉到什么？安全？掌控？兴奋？这是设计的核心指引。" |
| 核心隐喻 | "如果用一个现实世界的物体来比喻这个产品，会是什么？例如'驾驶舱'、'实验台'、'画廊'。" |
| OMTM | "如果只能用一个指标来衡量这个项目的成功，会是什么？口径怎么算？目标是多少？" |
| 主终端 | "这个产品主要在哪个终端使用？Web、Mobile、Landing 还是 Desktop？" |
| 首要动作 | "用户进入产品后，第一个想做的动作是什么？点击哪里？" |

### 优先级 2（强烈建议追问 - 但可先推进）

| 缺失字段 | 追问话术 |
|---|---|
| 成功瞬间 | "有没有一个'高光时刻'让用户觉得'值了'？比如任务完成时的动画、数据可视化出现的瞬间。" |
| In Scope / Out of Scope | "这次迭代明确要做什么？明确不做什么？我需要避免范围蔓延。" |
| 信息密度目标 | "这个产品应该信息密集（如交易终端）还是留白宽松（如冥想 App）？" |
| 核心用户流程 | "用户完成一个任务的主要步骤是什么？可以简单描述一下。" |
| 技术栈 | "前端技术栈有要求吗？比如 React、Vue、或者特定框架？" |

### 优先级 3（可选追问 - 不阻塞）

| 缺失字段 | 追问话术 |
|---|---|
| 数据与埋点 | "有需要埋点统计的关键行为吗？没有的话可以先跳过，后续补充。" |
| 发布策略 | "发布计划有要求吗？没有的话可以先跳过。" |

## 三、判断流程

```
PRD 输入
    │
    ▼
┌─────────────────────────────┐
│ 检查 P0 必填字段            │
│ (1.1 必填字段表)            │
└─────────────────────────────┘
    │
    ├─ 全部存在 ──────────────────┐
    │                            ▼
    │               ┌─────────────────────────────┐
    │               │ 检查 P1 强烈建议字段        │
    │               │ (1.2 强烈建议字段表)        │
    │               └─────────────────────────────┘
    │                            │
    │                            ├─ 存在缺失 ── 输出警告（不阻塞）
    │                            │
    │                            ▼
    │               ┌─────────────────────────────┐
    │               │ 返回：PRD 完备，可进入下游  │
    │               └─────────────────────────────┘
    │
    └─ 存在缺失 ─────────────────┐
                               ▼
               ┌─────────────────────────────┐
               │ 按追问优先级表生成追问列表  │
               │ 优先级 1 字段优先追问      │
               └─────────────────────────────┘
                               │
                               ▼
               ┌─────────────────────────────┐
               │ 返回：缺失字段列表 + 追问话术│
               │ 等待用户补充                │
               └─────────────────────────────┘
```

## 四、与 `validate_prd_completeness` skill 的映射

| 失败码 | 对应条件 | 返回内容 |
|---|---|---|
| `E031` | P0 字段缺失 | `{ missing_fields: [...], priority: 1, ask_prompts: [...] }` |
| (警告) | P1 字段缺失 | `{ warnings: [...], can_proceed: true }` |

---

## 五、示例输出

### 示例 1：P0 字段缺失

```json
{
  "passed": false,
  "missing_fields": [
    { "field": "目标用户是谁", "section": "1.2", "priority": 1 },
    { "field": "情绪目标", "section": "1.3", "priority": 1 }
  ],
  "ask_prompts": [
    "这个产品是给谁用的？我需要知道他们的身份才能设计正确的体验。",
    "你希望用户在使用后感觉到什么？安全？掌控？兴奋？这是设计的核心指引。"
  ],
  "can_proceed": false
}
```

### 示例 2：P1 字段缺失（可继续）

```json
{
  "passed": true,
  "warnings": [
    { "field": "成功瞬间", "section": "1.3", "priority": 2, "suggestion": "建议补充高光时刻设计指引" },
    { "field": "In Scope / Out of Scope", "section": "2.3", "priority": 2, "suggestion": "建议明确范围避免蔓延" }
  ],
  "can_proceed": true
}
```

### 示例 3：完备

```json
{
  "passed": true,
  "warnings": [],
  "can_proceed": true,
  "message": "PRD 完备，可进入 UX/Visual/Dev 下游"
}
```
