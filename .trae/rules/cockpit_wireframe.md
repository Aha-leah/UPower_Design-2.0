---

## 二、Header 区域

### 2.1 组成元素

| 元素 | 位置 | 说明 |
|---|---|---|
| 项目名 | 左侧 | 当前项目名称，可点击切换 |
| 当前阶段 | 中间偏左 | `正在执行 build...` |
| 进度条 | 中间 | 0-100%，可视化进度 |
| 设置按钮 | 右侧 | 项目设置、团队配置 |

### 2.2 进度条样式

```
[████████████░░░░░░░░] 65%
```

- 绿色：已完成
- 灰色：未完成
- 动画：执行中时进度条流动

---

## 三、Left Chat 区域（40% 宽度）

### 3.1 组成元素

| 区域 | 高度占比 | 说明 |
|---|---|---|
| 消息历史 | 70% | 用户与 Agent 的对话记录 |
| 工具调用日志 | 20% | 当前执行的工具/步骤 |
| 输入框 | 10% | 用户输入区域 |

### 3.2 消息历史

```
┌──────────────────────────────────────┐
│ 👤 用户                              │
│ "帮我创建一个任务管理工具"            │
│                              10:30  │
├──────────────────────────────────────┤
│ 🤖 Atlas                             │
│ 好的，我来帮你定义这个产品。          │
│                                      │
│ 我理解你需要：                        │
│ 1. 一个轻量但有结构的任务管理工具     │
│ 2. 支持 5-20 人团队                  │
│ 3. 渐进式复杂度                       │
│                              10:31  │
├──────────────────────────────────────┤
│ 👤 用户                              │
│ 对，就是这个方向                      │
│                              10:32  │
└──────────────────────────────────────┘
```

### 3.3 工具调用日志

```
┌──────────────────────────────────────┐
│ ⚙️ 执行日志                          │
├──────────────────────────────────────┤
│ ✅ load_context         10:32:15    │
│ ✅ generate_prd         10:33:42    │
│ ⏳ generate_brand_dna   10:35:00    │

```

**状态图标**：
- ✅ 成功
- ⏳ 执行中
- ❌ 失败
- ⏸️ 等待

### 3.4 输入框

```
┌──────────────────────────────────────┐
│ 在此输入指令...                  [发送]│
└──────────────────────────────────────┘
```

## 四、Right Canvas 区域（60% 宽度）

### 4.1 组成元素（从上到下）

| 面板 | 高度占比 | 说明 |
|---|---|---|
| 产物面板 | 40% | 已生成/待生成的文件列表 |
| 证据面板 | 30% | Reviewer 报告、验证结果 |
| 风险面板 | 30% | Stop Conditions、错误、警告 |

### 4.2 产物面板

```
┌──────────────────────────────────────┐
│ 📦 产物                              │
├──────────────────────────────────────┤
│ ✅ 📄 prd(input).md      12KB  10:33│
│ ✅ 📄 brand_dna.md        8KB  10:35│
│ ⏳ 🎨 style_prompt.md     -    执行中│
│ ⏸️ 📐 specs.md            -    等待  │
│ ⏸️ 🎬 motion.md           -    等待  │
└──────────────────────────────────────┘
```

**交互**：
- 点击文件名 → 预览/跳转
- 右键 → 下载/删除/重新生成

### 4.3 证据面板

```
┌──────────────────────────────────────┐
│ 📋 证据                              │
├──────────────────────────────────────┤
│ ┌────────────────────────────────────┐
│ │ 🔍 Visual Review       ❌ 1 issue │
│ │ 对比度不足：首页标题                │
│ │                    [查看详情]       │
│ └────────────────────────────────────┘
│ ┌────────────────────────────────────┐
│ │ 🔍 Interaction Review  ✅          │
│ │ 所有交互检查通过                    │
│ └────────────────────────────────────┘
│ ┌────────────────────────────────────┐
│ │ 📊 PRD 完备性          ✅          │
│ │ 所有必填字段已填写                  │
│ └────────────────────────────────────┘
└──────────────────────────────────────┘
```

**交互**：
- 展开查看详情
- 点击问题 → 跳转到相关代码位置

### 4.4 风险面板

```
┌──────────────────────────────────────┐
│ ⚠️ 风险                              │
├──────────────────────────────────────┤
│ ┌────────────────────────────────────┐
│ │ 🔴 需要确认                         │
│ │ 情绪目标不明确                      │
│ │                                    │
│ │ 你希望用户感觉到什么？              │
│ │ [安全] [掌控] [轻盈] [其他...]      │
│ └────────────────────────────────────┘
│                                      │
│ ┌────────────────────────────────────┐
│ │ 🟡 警告                             │
│ │ PRD 缺少成功瞬间描述                │
│ │ [忽略] [补充]                       │
│ └────────────────────────────────────┘
└──────────────────────────────────────┘
```

**交互**：
- Stop Conditions 显示确认按钮
- 警告可选择忽略或修复
- 错误显示重试按钮

## 五、响应式设计

### 5.1 断点策略

| 屏幕宽度 | 布局 |
|---|---|
| ≥1440px | Left 40% / Right 60% |
| 1024-1439px | Left 45% / Right 55% |
| 768-1023px | 上下布局（Chat 上 / Canvas 下） |
| <768px | Tab 切换（Chat / Canvas） |

### 5.2 移动端布局

```
┌──────────────────────────┐
│ [Chat] [Canvas] [更多]   │ ← Tab 切换
├──────────────────────────┤
│                          │
│   当前 Tab 内容          │
│                          │
│                          │
└──────────────────────────┘
```

---

## 六、交互规范

### 6.1 拖拽调整

- Left/Right 区域宽度可拖拽调整
- 最小宽度：Left 300px，Right 400px

### 6.2 快捷键

| 快捷键 | 功能 |
|---|---|
| `Cmd/Ctrl + Enter` | 发送消息 |
| `Cmd/Ctrl + 1` | 切换到 Chat |
| `Cmd/Ctrl + 2` | 切换到 Canvas |
| `Esc` | 关闭详情弹窗 |

### 6.3 通知

| 事件 | 通知方式 |
|---|---|
| 需要用户确认 | 弹窗 + 声音提示 |
| 阶段完成 | Toast 通知 |
| 错误发生 | 红色 Banner |

---

## 七、实现建议

### 7.1 技术栈

- **前端框架**：React 18
- **布局**：Tailwind CSS Flexbox
- **状态管理**：Zustand
- **实时更新**：WebSocket 或 Server-Sent Events
